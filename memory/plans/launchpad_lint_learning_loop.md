# Launchpad Lint Learning Loop

## Goal

Define how real launch results should change the factory.

## After The First Launch Test

Capture:

- install count
- first paid call count
- full two-call session count
- user-reported clarity gain
- major objections or failure notes

## Interpretation Rules

### Good launch signal

- at least one clean two-call activation
- clear user statement that the product reduced launch ambiguity
- output needed only light editing

Action:

- keep the lane alive
- let `growth` broaden visibility carefully
- let `pricing` observe price sensitivity before changing pricing

### Mixed signal

- users engage, but the output still needs heavy rewriting

Action:

- send concrete examples to `designer` and `builder`
- tighten the product before expanding distribution

### Weak signal

- installs happen, but users do not complete the two-call workflow

Action:

- inspect whether the problem is positioning, activation friction, or tool ordering
- do not treat installs as evidence of demand

### Bad signal

- no meaningful activation and no strong qualitative evidence of usefulness

Action:

- kill or pause the lane
- update `researcher` rankings and candidate selection logic

## Output Artifact

After the first launch cycle, create a post-launch valuation note comparing:

- predicted activation
- observed activation
- predicted value
- observed value
- recommended next action
